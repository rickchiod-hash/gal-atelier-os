# Engineering Standards

## Backend

- Kotlin com data classes e value object `Money`.
- UseCase por intenção.
- Ports para saída.
- Logs com traceId.
- Erros padronizados.
- Testes unitários para domínio e aplicação.

## Frontend

- Design tokens.
- Responsivo.
- Form acessível.
- Sem regra financeira no client.
- Status de operação claro.
